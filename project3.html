Implementing a CI/CD Pipeline for a React Application using Jenkins and Docker on Ubuntu EC2 Instances

  • Source code

Project Overview:

The project involves setting up a Continuous Integration/Continuous Deployment (CI/CD) pipeline for a React application using Jenkins and Docker. The infrastructure includes two Ubuntu EC2 instances, one hosting the Jenkins server and the other hosting Docker.

Jenkins Server Setup

The first step involves setting up the Jenkins server on one of the EC2 instances. This includes installing Jenkins and necessary plugins like the Git plugin and Docker plugin. Jenkins is used to automate the parts of software development related to building, testing, and deploying, facilitating continuous integration and continuous delivery.

Docker Server Setup:

The second EC2 instance is set up with Docker. Docker is a platform used to develop, ship, and run applications inside containers. It provides a way to package applications with their dependencies into a standardized unit for software development.

React Application and Docker:

The React application is created using the npx create-react-app command. A Dockerfile is created for the React app, which contains instructions to build a Docker image of the app. The Docker image includes everything needed to run the application, including the code, runtime, libraries, and system tools.

Jenkins Pipeline:

A Jenkinsfile is created to define the Jenkins pipeline. The pipeline checks out the code, builds the Docker image, and deploys it. The Jenkinsfile uses the Docker Hub credentials stored in Jenkins to log in to Docker Hub, push the Docker image, and then pull the image on the Docker server and run it.

Conclusion:

This project demonstrates the power of CI/CD pipelines in automating the software development process, making it more consistent and efficient. It also highlights the use of Docker in creating a consistent environment for the application, from development to production. The use of Jenkins allows for the automation of various stages in the development lifecycle, including building, testing, and deployment.
